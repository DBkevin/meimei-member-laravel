# Stage 2：积分兑换业务闭环

## 一、本阶段目标

让系统完成一次完整的会员积分兑换商品、后台核销、积分和库存同步变化的业务闭环。

## 二、已完成内容

1. 会员创建后自动生成积分账户
2. 积分账户支持增加、扣减、退还等操作
3. 商品兑换时校验积分、库存、商品状态
4. 兑换成功后扣减会员积分
5. 兑换成功后扣减商品库存
6. 兑换成功后生成兑换订单
7. 后台支持核销、取消、拒绝订单
8. 核销订单后生成核销记录
9. 取消或拒绝订单后退还积分、恢复库存

## 三、核心服务说明

`app/Services/PointRedemptionService.php`

主要方法：
- `redeem` - 兑换商品
- `verify` - 核销订单
- `cancel` - 取消订单
- `reject` - 拒绝订单

## 四、订单状态流转

**待处理订单（pending）**可以进入三种结果：

1. **核销订单** → 生成核销记录
2. **取消订单** → 退还积分、恢复库存
3. **拒绝订单** → 退还积分、恢复库存

**已核销订单（verified）**：
- 不允许再次取消
- 不允许再次拒绝
- 不允许重复核销

**已取消订单（cancelled）**：
- 不允许再次核销

**已拒绝订单（rejected）**：
- 不允许再次核销
