# Stage 2：积分兑换业务闭环 - 阶段验收文档

## 一、本阶段目标

实现"会员积分兑换商品， 并由后台核销"的完整业务闭环。

## 二、已实现业务闭环

### 1. 会员创建后自动生成积分账户
- 创建 Member 时自动创建 PointAccount
- 使用 firstOrCreate 避免重复创建

### 2. 积分流水影响积分账户余额
- earnPoints: 增加积分，余额增加
- spendPoints: 扣减积分，余额减少，禁止扣成负数
- adjustPoints: 调整积分（管理员操作）
- refundPoints: 退款积分（取消订单时退回）
- freezePoints: 冻结积分
- unfreezePoints: 解冻积分
- 每次积分变动自动生成 PointTransaction

### 3. 积分兑换商品流程
- 校验会员存在
- 校验会员积分账户存在
- 校验商品已上架
- 校验商品库存充足
- 校验会员可用积分充足
- 兑换成功后创建 RedemptionOrder
- 兑换成功后扣减会员积分
- 兑换成功后扣减商品库存
- 整个流程使用数据库事务

### 4. 订单状态流转
- pending: 待处理
- verified: 已核销
- cancelled: 已取消
- rejected: 已拒绝
- 只允许 pending 状态流转到其他状态

### 5. 订单核销后生成核销记录
- 核销时自动生成 VerificationRecord
- 记录订单、会员、核销人、核销时间、备注

## 三、核心服务说明

### PointRedemptionService
- redeem(): 积分兑换商品
- verify(): 核销订单
- cancel(): 取消订单（退还积分、恢复库存）
- reject(): 拒绝订单（退还积分、恢复库存）
